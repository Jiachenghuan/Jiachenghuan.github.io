## Why many course notes are never opened again {#why-notes-fail}
They preserve the order of the lecture but fail to preserve the relationships between the ideas.
## First capture what was taught, then extract what should remain {#what-to-capture}
### Layer one: facts and concepts {#facts-and-concepts}
Start by keeping the key terms, definitions, and examples. This is the raw input layer.
### Layer two: relations and boundaries {#relations-and-boundaries}
The more important work is to understand how concepts explain each other, where they stop applying, and what problem each one is really solving.
## Compress one session into a few sharp questions {#compress-questions}
If a class can be reduced to three to five core questions, the notes become much easier to revisit and reuse.
## Let notes become the draft for the next output {#turn-into-output}
A strong set of course notes can be turned directly into a recap, a talk outline, or an article skeleton. A note is only finished when it enters an output path.
